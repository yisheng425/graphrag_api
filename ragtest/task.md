
子任务4.2.1.1 多模态实体识别与关系抽取：​
统一规范与样本集 ​
来自不同模态（文本、图像、语音）的标注/标签 → 统一命名、统一格式化。​
形成可统一检索与处理的多模态样本集。​
设计 Schema / 概念体系（先前基本已完成）​
层级结构：“产品信息 — 故障模式 — 伤害情形 — 召回事件”。 ​
明确每层实体、关系的属性定义。 ​
大模型辅助的实体映射​
根据不同模态，按照实施方案的技术路线进行抽取。​
把这些抽取的实体对齐到 Schema 中已有的类型/实例，如果已有则合并，如果没有则新增。​
候选关系生成 + 大模型辅助精化​
基于 Schema 里定义的语义关系生成候选对（如“产品信息–存在故障–故障模式”）。​
大模型辅助进行更精细分类，筛选出合理关系，得到语义三元组。​
三元组入库​
将最终的 <实体, 关系, 实体> 写入图数据库，逐步构建产品安全知识图谱。
